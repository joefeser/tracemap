# Requirements: Swift API-Client Evidence Walkthrough

## Goal

Add a public-safe demo page that explains how to read Swift API-client
evidence as static rule-backed candidates.

## Requirements

- The page must live at `/swift/api-client-walkthrough/`.
- Public claim level must be `demo`.
- The page must explain URLSession, URLRequest, Alamofire, and Moya-style
  evidence as static candidates only.
- Rule IDs, evidence tier, coverage label, limitations, and non-claims must
  stay attached to the claim.
- The page must link to `/swift/surface-discovery/`,
  `/swift/real-world-smoke/`, `/swift/claim-language/`, and validation docs.
- The page must avoid endpoint reachability, backend compatibility, request
  success, auth flow, production traffic, runtime behavior, API correctness,
  complete Swift semantic analysis, and AI-analysis claims.
- Site validation must cover route metadata, sitemap/discovery entries, required
  evidence-shape rows, required links, and safety wording.
