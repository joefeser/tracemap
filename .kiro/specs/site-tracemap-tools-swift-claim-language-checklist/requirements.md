## Intent

Publish a reviewer-facing Swift v0 claim-language checklist so public Swift copy
can be repeated without drifting past TraceMap's static evidence boundary.

## Requirements

- Add a public route for Swift claim-language review.
- Anchor the page to Swift v0 on `main`, PR #425, and public proof routes.
- Make the checklist discoverable from the Swift evidence lane, Swift story,
  real-world smoke proof, and Swift evidence safety page.
- Require public claim level, proof path, evidence tier, coverage label,
  limitation, and non-claim boundary before repeating Swift copy.
- Include allowed wording examples for shipped Swift v0, demo smoke evidence,
  and reduced-coverage gaps.
- Block runtime/API correctness, simulator/device, Xcode build, SwiftPM restore,
  production, package compatibility, release safety, storage/runtime, complete
  Swift semantic analysis, and AI-analysis claims.
- Keep raw generated artifacts, source snippets, raw SQL, secrets, credentials,
  local absolute paths, raw remotes, and private scan artifacts out of public copy.
