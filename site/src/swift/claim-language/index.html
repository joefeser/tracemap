Swift reviewer checklist

Repeat Swift claims only when the evidence label travels with them.

Swift v0 is shipped as static evidence discovery. This checklist keeps public copy tied to proof paths, rule-backed evidence, coverage labels, limitations, and non-claims before a reviewer repeats it.

Public claim level: shipped. Proof: PR #425. The checklist supports public wording review; it does not upgrade static evidence into runtime API correctness or complete Swift semantic analysis.

Blocked wording

Do not let stronger product language sneak in through Swift copy.

  • Do not claim runtime behavior, runtime API correctness, endpoint reachability, backend compatibility, app navigation, rendered UI, user action, production usage, deployment state, release safety, stored values, query execution, live schema, or complete Swift semantic analysis.
  • Do not claim Xcode build proof, SwiftPM restore proof, simulator/device execution proof, package compatibility, dependency freshness, dependency license status, vulnerability status, or operational safety.
  • Do not describe TraceMap scanner or reducer work as AI impact analysis, LLM analysis, prompt-based classification, embedding-backed search, or vector database analysis.
  • Do not publish raw generated artifacts, raw source snippets, raw SQL, secrets, credentials, local absolute paths, raw remotes, stored values, private scan artifacts, analyzer logs, or hidden validation details.
