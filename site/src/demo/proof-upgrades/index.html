Claim boundary

What this upgraded demo can and cannot support.

  • Safe: the checked-in public demo can generate these six report families and summary rows.
  • Safe: the rows are static, rule-backed, evidence-tiered, coverage-labeled, and limited by visible gaps.
  • Safe: reviewers can inspect generated public-safe Markdown and JSON reports for sample evidence trails.
  • Not safe: claiming full impact, runtime topology, release approval, production dependency understanding, endpoint performance, or production usage.
  • Not safe: publishing raw source snippets, raw SQL, config values, secrets, local absolute paths, raw repository remotes, facts, SQLite indexes, combined SQLite files, or analyzer logs.

This is an evidence upgrade, not a louder promise.

The stronger site claim is narrow: these six rows moved from deferred to demo because the public demo now produces reproducible, public-safe evidence from checked-in samples. Shipped production claims still require the relevant capability, proof path, and limitations to be true on main.
