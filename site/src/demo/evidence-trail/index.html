How to read it

Rule IDs, evidence tiers, coverage labels, and limitations travel together.

Rule IDsIdentify the deterministic extractor or summary rule behind a row.
Evidence tiersName confidence in the static evidence: Tier1Semantic, Tier2Structural, Tier3SyntaxOrTextual, or Tier4Unknown.
Coverage labelsKeep complete, reduced, partial, missing, or unknown evidence visible.
LimitationsExplain what the packet cannot support from the selected proof source.

Claim boundary

The trail is browsable evidence, not a stronger conclusion.

  • Safe: the checked-in public summary shows a changed-surface count, endpoint findings, path counts, gaps, rule ID, evidence tier, coverage label, and public-safe report-family paths.
  • Safe: downstream package, configuration, and SQL-facing rows are named as gaps when the selected public summary lacks per-item evidence.
  • Not safe: treating the walkthrough as runtime proof, production proof, endpoint-performance proof, outage cause, operational assurance, release approval, or complete product coverage.
  • Not safe: publishing private identifiers, generated local internals, database text, configuration data, source excerpts, credentials, or workstation paths.
