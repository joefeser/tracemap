Proof path index

Find the public evidence trail before repeating the claim.

This index maps TraceMap public pages and demo sections to the static evidence surfaces that support them: artifact type, rule ID or rule family, evidence tier, coverage label, proof path, limitation, and public status.

Public claim level: demo. Shared site principle: No public conclusion without evidence. Page-level demo status and per-entry public status are separate axes.

How to use it

The same evidence row should answer four readers without changing the claim.

ManagersStart with public status, coverage label, limitation, and the public-safe route that can be reviewed without raw scan internals.
ReviewersCheck the rule ID or rule family, evidence tier, coverage label, and proof path before approving wording.
EngineersFollow the scripts, samples, generated summary fixture, and public reports that reproduce or support the row.
BotsUse stable labels such as demo, future, Tier2Structural, PartialAnalysis, and route paths for link and claim checks.

Artifact vocabulary

Public pages can name raw artifacts without publishing raw internals.

scan-manifest.jsonLocal scan identity, repo and commit metadata, scanner versions, and coverage context. Public only when a sanitized sample explicitly checks it in.
facts.ndjsonLocal fact stream with rule-backed evidence rows. Public pages may summarize counts and labels, not publish private raw facts.
index.sqliteLocal queryable index for symbols, endpoints, relationships, surfaces, paths, and gaps. Private SQLite files stay local-only.
report.mdHuman-readable scan report. Public-safe only when generated from public samples and reviewed for raw values.
logs/analyzer.logLocal diagnostic log for extraction and coverage troubleshooting. It is not public discovery content.
demo-summary.mdPublic-safe generated summary with statuses, rule IDs, evidence tiers, coverage labels, counts, and reasons.
demo-summary.jsonMachine-readable public-safe summary fixture used by demo pages and validation checks.
reports/**/*.mdPublic-safe report families only when produced from public samples and scrubbed by generated-output checks.
sitemap metadataThis standalone route is included in site/src/_site/pages.json; discovery metadata remains a route index, not proof by itself.

Non-claims

The index keeps the no-claims visible.

  • No runtime proof, production traffic proof, endpoint performance proof, deployment state proof, release safety proof, or release approval proof.
  • No AI impact analysis, LLM calls, embeddings, vector databases, or prompt-based classification in TraceMap core scanner or reducer claims.
  • No raw facts, SQLite files, analyzer logs, source snippets, raw SQL, config values, secrets, local absolute paths, raw repository remotes, generated scan directories, or private sample identities.
  • No parity claim across language adapters, workflows, samples, or report types unless the cited public-demo evidence supports that specific row.
